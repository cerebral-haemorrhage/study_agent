# Agent 开发学习路线

> 学习方式：**先讲原理 → 手搓版模拟 → 框架版实践 → 对比两者差异**

---

## 第一层：工具 (Tool System) ✅ 已完成

| 知识点 | 学到的 |
|--------|--------|
| 多工具共存 | `@tool` 装饰器，`bind_tools`，`TOOLS` 列表管理 |
| 工具错误处理 | `try/except`，异常不崩溃，返回错误信息给 LLM |
| 参数校验 | 空值检查，引导 LLM 反向请求用户补充 |
| Fallback 机制 | 主库 → 备库 → 兜底三级查询 |

---

## 第二层：记忆 (Memory) 🔄 进行中

| 知识点 | 要解决的问题 |
|--------|------------|
| **短期记忆 + MemorySaver** | 当前对话怎么记？跨轮对话自动恢复（已学） |
| **记忆窗口** | 对话太长超过 token 上限，只保留最近 N 条消息 |
| **摘要记忆** | 把旧对话压缩成一句话摘要，节省 token |
| **SqliteSaver** | 关程序重启后还能记住之前聊了什么（替代 MemorySaver） |

---

## 第三层：MCP 协议 (Model Context Protocol) 🔥

| 知识点 | 要解决的问题 |
|--------|------------|
| **MCP 原理** | 什么是 MCP？为什么 Anthropic/OpenAI 都在推？ |
| **MCP Server** | 把 `@tool` 改造成标准 MCP Server，任何客户端都能连 |
| **MCP Client** | LangGraph Agent 通过 `langchain-mcp-adapters` 连 MCP Server |

---

## 第四层：推理 + 结构化输出

| 知识点 | 要解决的问题 |
|--------|------------|
| **ReAct** | 思考 → 行动 → 观察 循环（已学） |
| **Plan-Execute** | 先制定计划，再逐步执行 |
| **Structured Output** | 强制 LLM 返回 JSON，不再依赖 prompt 约定格式 |
| **反思模式** | 执行完对自己结果做自我检查 |

---

## 第五层：流式交互 + Human-in-the-loop

| 知识点 | 要解决的问题 |
|--------|------------|
| **Streaming 流式输出** | 像 ChatGPT 那样一个字一个字蹦出来 |
| **Human-in-the-loop** | 敏感操作前暂停，等人类确认 |
| **中断与恢复** | 跑一半中断了，从断点继续 |

---

## 第六层：RAG（检索增强生成）

| 知识点 | 要解决的问题 |
|--------|------------|
| **向量数据库** | 怎么把文字变成"意思相近的向量"？ |
| **检索增强** | 先查资料 → 喂给 LLM → 基于资料回答 |
| **混合检索** | 关键词 + 语义搜索，取长补短 |

---

## 第七层：多 Agent 协作 + A2A

| 知识点 | 要解决的问题 |
|--------|------------|
| **路由 / 顺序 / 并行模式** | 多 Agent 怎么分工？ |
| **A2A 协议 (Agent-to-Agent)** | Google 推的 Agent 间通信标准 |
| **监督模式** | 主管 Agent 分配任务，验收结果 |

---

## 第八层：评测 + 生产部署 🔥

| 知识点 | 要解决的问题 |
|--------|------------|
| **LangSmith 追踪** | 每一步花了多少 token、多少时间？ |
| **评测体系** | Agent 回答好不好？准确率/延迟/成本怎么衡量？ |
| **Guardrails** | Agent 胡说八道/乱调工具的安全护栏 |

---

## 学习顺序

```
工具深化 ✅ → 记忆体系 🔄 → MCP → 推理+结构化 → 流式交互 → RAG → 多Agent+A2A → 评测部署
```
