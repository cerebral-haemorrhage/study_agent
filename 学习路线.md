# Agent 开发学习路线

> 学习方式：**先讲原理 → 手搓版模拟 → 框架版实践 → 对比两者差异**

---

## 第一层：工具 (Tool System)

**当前状态：** 只有 `get_weather` 一个工具，太单薄

| 知识点 | 要解决的问题 |
|--------|------------|
| **多工具共存** | 当 agent 有 5 个、10 个工具时，怎么组织？LLM 怎么选？ |
| **工具错误处理** | 工具执行失败了怎么办？agent 要不要重试？ |
| **参数校验** | 用户说"查一下天气"，没提供城市名，怎么办？ |
| **Fallback 机制** | 主工具挂了，自动切备用方案 |

---

## 第二层：记忆 (Memory)

**当前状态：** `ConversationMemory` 只活在内存里，关了程序就没了

| 知识点 | 要解决的问题 |
|--------|------------|
| **短期记忆** | 当前对话怎么记？（已学） |
| **持久化存储 (Checkpointer)** | 关了重开还能记住之前聊了什么 |
| **长期记忆** | 跨会话的记忆，比如"用户之前说过他叫小明" |
| **摘要记忆** | 对话太长超过 token 上限，怎么压缩总结？ |
| **记忆窗口** | 只保留最近 N 轮，防止上下文爆炸 |

---

## 第三层：推理 (Reasoning)

**当前状态：** 只学了 ReAct 一种推理模式

| 知识点 | 要解决的问题 |
|--------|------------|
| **ReAct** | 思考 → 行动 → 观察 循环（已学） |
| **Plan-Execute** | 先制定计划，再逐步执行，无需重复思考 |
| **ReWOO** | 一次性生成所有工具调用，并行执行 |
| **反思模式 (Reflection)** | 执行完对自己结果做自我检查 |
| **伪代码推理** | 让 LLM 先写伪代码再执行，处理复杂逻辑 |

---

## 第四层：交互 (Interaction)

**当前状态：** "黑盒等待 → 一次性吐出答案"

| 知识点 | 要解决的问题 |
|--------|------------|
| **Streaming 流式输出** | 像 ChatGPT 那样一个字一个字蹦出来，不要干等 |
| **Human-in-the-loop** | 执行敏感操作前暂停，等人类点"确认" |
| **中断与恢复** | Agent 跑一半被中断了，下次能从断点继续 |
| **中间结果展示** | 不只是最终答案，每一步在想什么都要让用户看到 |

---

## 第五层：知识 (Knowledge / RAG)

**当前状态：** Agent 只能用工具获取实时数据，没有"已学知识"

| 知识点 | 要解决的问题 |
|--------|------------|
| **文档加载与拆分** | 怎么把 PDF、网页、数据库内容变成 agent 能搜的格式？ |
| **向量数据库** | 怎么把文字变成"意思相近的向量"？ |
| **检索增强 (RAG)** | 先查资料库 → 把相关资料喂给 LLM → 基于资料回答 |
| **混合检索** | 关键词搜索 + 语义搜索，取长补短 |

---

## 第六层：多 Agent 协作 (Multi-Agent)

**当前状态：** 一个 Agent 干所有活

| 知识点 | 要解决的问题 |
|--------|------------|
| **路由模式** | 根据用户意图分发给不同 Agent（客服/技术/销售） |
| **顺序协作** | Agent A 处理完 → 交给 Agent B → 再交给 Agent C |
| **并行协作** | 多个 Agent 同时分析同一问题，最后投票或汇总 |
| **监督模式** | 一个主管 Agent 分配任务给下属 Agent，验收结果 |
| **Agent 间通信** | Agent 之间怎么互相传数据、发消息？ |

---

## 推荐学习顺序

```
工具深化 → 记忆体系 → 推理模式 → 流式交互 → RAG → 多Agent
  (现在)     ↓          ↓         ↓        ↓       ↓
           打好     学会多种   用户体验  连接外部  规模化
          基础      思考方式              知识
```
